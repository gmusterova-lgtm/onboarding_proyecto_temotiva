ONBOARDING GENERAL

👋 BIENVENID@

Bienvenida/o a Temotiva. Este documento es para que cualquier persona que se incorpore al equipo entienda rápido dónde entra, cómo trabajamos y qué tiene que hacer durante sus primeros días.

La idea no es que memorices todo, sino que tengas una guía clara para empezar con orden, preguntar mejor y aportar valor cuanto antes.

Temotiva es un proyecto de bienestar emocional preventivo. Combinamos app, psicología aplicada, IA, talleres, investigación, comunidad y producto digital para ayudar a las personas a cuidarse mejor en su vida cotidiana.

1. Entiende el proyecto antes de empezar

Antes de tocar una tarea concreta, entiende qué estamos construyendo. Temotiva no es solo una app. Es un proyecto en crecimiento, con usuarios reales, validación en marcha, talleres, datos, decisiones de producto y una visión clara: ayudar a las personas a identificar cómo se sienten, anticipar señales de malestar y desarrollar hábitos de autocuidado.

2. Ubícate en tu área

Este documento es general. Sirve para todas las personas que entran en Temotiva, pero no sustituye la documentación específica de cada área.

Después de leerlo, navega en el menú superior a la pestaña que corresponda a tu rol. Ahí encontrarás las tareas, procesos, criterios, responsables, herramientas y materiales propios de tu área. Si no sabes cuál es tu pestaña, no improvises. Pregunta primero a tu responsable directo.

3. Ten claro tu rol

Al entrar, necesitas saber tres cosas básicas: qué rol ocupas, qué se espera de ti y quién revisa tu trabajo. No empieces tareas sueltas sin saber para qué sirven.

Primero confirma cuál es tu prioridad, qué resultado se espera y cómo debes reportarlo. En Temotiva valoramos la autonomía, pero la autonomía no significa ir por libre. Significa entender el objetivo, avanzar con criterio y comunicar bien.

4. Revisa tus accesos

Comprueba que tienes acceso a las herramientas necesarias para tu trabajo, y solo a las necesarias (Discord, Notion, Drive, GitHub, etc.). Si te falta un acceso, pídelo indicando qué herramienta necesitas, para qué tarea concreta y si es urgente. No pidas accesos "por si acaso".

5. Trabaja con mentalidad de startup

Temotiva está creciendo, así que no todo estará perfecto, cerrado o documentado desde el primer día. Aquí necesitamos personas que sepan avanzar con lo que hay, ordenar lo que falta, avisar cuando algo bloquea y proponer soluciones razonables.

6. Comunica

Comunicar bien no significa escribir mucho. Significa que la otra persona entienda rápido qué pasa. Cuando reportes avances, deja claro qué hiciste, qué resultado hubo, qué queda pendiente, qué problema encontraste y cuál es el siguiente paso.

Si estás bloqueada/o, dilo pronto. Un bloqueo comunicado a tiempo se puede resolver; un bloqueo oculto retrasa al equipo.

7. Documenta lo importante

En Temotiva, documentar también es trabajar. Si haces un proceso, cambias algo, detectas un error o aprendes algo útil, déjalo escrito. La documentación no tiene que ser perfecta, pero sí permitir que otra persona entienda el proceso. Un proyecto crece mejor cuando el conocimiento no se queda solo en una cabeza.

8. Entrega trabajo revisable

Todo lo que entregues debe poder revisarse. Antes de decir que una tarea está terminada, comprueba que se entiende, tiene sentido, está ordenada y otra persona podría continuar desde ahí. No buscamos perfeccionismo puro, buscamos trabajo útil, claro y mejorable.

9. Pide ayuda de forma responsable

Pedir ayuda está bien. Lo que no ayuda es preguntar sin contexto. Antes de preguntar, revisa la documentación disponible, mensajes anteriores y ejemplos. Cuando preguntes, explica qué intentabas hacer, qué probaste, dónde te bloqueaste y qué necesitas exactamente.

10. Cuida la información interna

Durante tu participación tendrás acceso a información interna. No compartas fuera del equipo documentos, datos, accesos, estrategias, conversaciones ni materiales no publicados sin autorización. La confianza es parte del trabajo.

11. Recibe feedback y aplícalo

En una startup se corrige, se ajusta y se mejora constantemente. Puede que se te pidan cambios o mejoras. No es algo personal, es parte de construir un buen producto. Recibir feedback, entenderlo y aplicarlo rápido es una habilidad muy valiosa.

12. Qué hacer durante tus primeros días

  1. Lee este onboarding completo.
  2. Entra en tu área correspondiente.
  3. Identifica a tu responsable directo.
  4. Revisa tus tareas iniciales.
  5. Confirma que tienes los accesos necesarios.
  6. Empieza por tareas pequeñas y bien documentadas.
  7. Reporta avances, dudas y bloqueos ordenadamente.

🔑 El archivo Secreto (.env)

La app necesita contraseñas (claves de Google, pagos) para funcionar. Por seguridad, estos datos nunca se suben a GitHub. Debes crear un archivo llamado .env en la raíz de tu proyecto.

Tip de Oro: Si usas un celular Android físico para probar la app, tu celular no entenderá qué es localhost. Debes poner la dirección IP local de tu internet (ej. 192.168.1.100). En el simulador de iPhone sí funciona localhost.

# Crea el archivo .env y pega esto:

# 1. API del Backend (Reemplaza con tu IP si usas Android físico)
EXPO_PUBLIC_API_URL=http://localhost:4000

# 2. Claves de Google (Pídelas a tu líder técnico)
EXPO_PUBLIC_GOOGLE_WEB_CLIENT_ID=...
EXPO_PUBLIC_GOOGLE_IOS_CLIENT_ID=...
EXPO_PUBLIC_GOOGLE_ANDROID_CLIENT_ID=...

# 3. Claves de Pagos (RevenueCat)
EXPO_PUBLIC_REVENUECAT_IOS_KEY=...
EXPO_PUBLIC_REVENUECAT_ANDROID_KEY=...

🚀 Guía Paso a Paso: Tu primer día

Sigue estas instrucciones al pie de la letra para arrancar la app en tu computadora:

Paso 1: Descarga el código y levanta el Backend (El Cerebro)

La app no sirve de nada sin el servidor que le da los datos. Haremos que tu PC simule el servidor usando Docker.

git clone https://github.com/TEMOTIVA-REACT/Temotiva-React.git

# 2. Cambia a la rama del servidor local
git checkout backend-dev

# 3. Enciende el servidor (Asegúrate que Docker Desktop esté abierto)
docker compose -f docker-compose.dev.yml up --build

Paso 2: Levanta la App Móvil (La Interfaz)

Abre otra pestaña nueva en tu terminal (no cierres la del paso 1 porque apagarías el servidor) y haz esto:

# 1. Cambia a la rama de desarrollo de la app
git checkout dev

# 2. Instala las librerías necesarias (solo se hace la primera vez)
npm install

# 3. (Asegúrate de haber creado tu archivo .env aquí)

# 4. Inicia la aplicación
npx expo start

Verás un código QR en tu terminal. Escanéalo con Expo Go en tu celular. (Si usas emulador, presiona 'i' para iOS o 'a' para Android en la terminal).

🚨 No entres en pánico (Errores Comunes)

Es normal que algo falle, a todos nos pasa. Revisa esta enciclopedia de errores antes de pedir ayuda:

Problemas con Git

Your branch is behind 'origin/dev' by X commits
Tu rama está desactualizada respecto al trabajo de tus compañeros. Baja los últimos cambios: 
git pull origin dev
error: failed to push some refs
Alguien subió código nuevo a GitHub mientras tú programabas. Descárgalo y combínalo: 
git pull origin <tu-rama> --rebase
git push origin <tu-rama>

Problemas con el Servidor (Backend/Docker)

port is already allocated (puerto 4000 ocupado)
Ya tienes algo corriendo en el puerto del backend. Búscalo y mátalo: 
lsof -i :4000
kill -9 <PID>
docker compose up
Cannot connect to the Docker daemon
Se te olvidó abrir la aplicación "Docker Desktop" en tu computadora. Ábrela y espera a que cargue.
relation "tabla" does not exist
Tu base de datos está vacía. Necesitas correr las migraciones del backend: 
docker compose exec backend npm run migration:run

Problemas con la App (Expo/React Native)

Network request failed / Cannot connect to API
¿Por qué pasa? Tu celular intenta buscar el backend en sí mismo (localhost) o Docker no ha terminado de encender.

Solución: Revisa que Docker diga "database system is ready". Si usas Android físico, cambia localhost por la IP de tu PC en el .env y reinicia con npx expo start --clear.
Unable to resolve module ...
Módulo faltante o caché corrupta porque alguien instaló una librería nueva. 
rm -rf node_modules
npm install
npx expo start --clear
Expo Go no escanea el QR
Tu PC y tu celular deben estar conectados al mismo Wi-Fi. Si sigue fallando, en la terminal de Expo presiona la tecla 's' para cambiar al modo Tunnel.
Apple Authentication failed
El inicio de sesión de Apple bloquea simuladores sin cuenta configurada. Para pruebas rápidas de desarrollo, usa siempre Google Sign-In.

EADDRINUSE: address already in use :::3000

kill -9 $(lsof -t -i:3000)

npm run dev
NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY is not defined
